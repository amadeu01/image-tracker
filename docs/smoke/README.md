# Manual smoke kit

`scripts/smoke-report.sh` runs the scriptable half of a release smoke test — no-args usage/exit code, a quick headless `track` run on the v3 test video (asserting the CSV has >= 2500 rows), and the version banner / log file check — then prints a GUI checklist (open a video, scrub, seed with `S`, calibrate with two clicks, watch `Track` run with a live crosshair, confirm the side panel shows guide/status/events, and pause+reseed+resume) for a human to walk through by hand. It writes both the auto-filled results and the empty GUI checkboxes to `docs/smoke/YYYY-MM-DD.md`, overwriting that day's report if run again. Run it with `./scripts/smoke-report.sh` from the repo root before cutting a release, then edit the generated report to check off the GUI items, fill in the tester/notes, and commit the file under `docs/smoke/`.
